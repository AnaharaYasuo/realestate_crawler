# 土地情報収集クローラー ＆ 高精度土地評価エンジン 詳細設計書

本ドキュメントは、戸建ておよびアパートの価格推定における土地価値評価の精度をプロの実務水準（不動産鑑定士・仕入デベロッパー水準）に引き上げるための、土地情報の収集仕様および評価エンジンの計算モデルに関する詳細設計です。

---

## 1. 土地情報の収集要件（クローラー拡張）

各不動産ポータルサイト（三井・住友・東急・野村・アットホーム・LIFULL HOME'S等）から新しく「売地（土地）」の物件情報を収集するための項目定義およびクローラー拡張方針を定義します。

### 1.1 収集必須とする土地固有パラメータ

| パラメータ名 | データベース物理名 | 型 | 説明 |
| :--- | :--- | :---: | :--- |
| **土地面積** | `tochiMenseki` | Dec(10,2) | 公簿または実測の土地面積（㎡）。必須。 |
| **間口（接道長）** | `maguchi` | Dec(4,2) | 道路に接している長さ（m）。接道義務・間口狭小補正に直結。 |

## 2. 土地情報の評価アルゴリズム設計

収集した土地データから、統計的および法的アプローチを組み合わせて客観的な土地価値を算出するロジックを設計します。

```
+-------------------------------------------------------------+
|                     土地総合評価エンジン                     |
+-------------------------------------------------------------+
                              |
       +----------------------+----------------------+
       |                      |                      |
       v                      v                      v
[1. 路線価・比準補正]   [2. 実質容積率の判定]   [3. 開発法・残余法]
- 不整形地補正          - 前面道路幅員制限      - 想定建築延床の算出
- 間口狭小・奥行長大    - 北側接道斜線緩和      - GDV - 建築費 = 土地価値
- 角地・二方路加算      - 容積消化率の算出      - 土地帰属純収益の還元
                              |
                              +----------------------+
                              |                      |
                              v                      v
                       [4. セットバック減価]   [5. 地盤・造成・環境リスク]
                       - 後退面積の算出除外    - 擁壁・がけ条例減価
                       - 有効土地面積の補正    - 土壌汚染・埋蔵文化財
```

### 2.1 鑑定・相続税評価実務に基づく個別的要因の比準（比準補正モデル）

国税庁の「財産評価基本通達」および不動産鑑定士が使用する「土地価格比準表」をシステム的なアルゴリズムに変換します。

#### ① 不整形地（かげ地）補正 ($C_{\text{shape}}$)
想定される「標準的な整形地（長方形）」の面積 $A_{\text{standard}}$ に対する、実際の土地の欠損部分（かげ地）の面積 $A_{\text{kage}}$ の比率（かげ地割合 $R_{\text{kage}}$）を算出します。
$$R_{\text{kage}} = \frac{A_{\text{standard}} - A_{\text{actual}}}{A_{\text{standard}}}$$
- **判定ロジック**:
  - $R_{\text{kage}} < 10\%$: 整形地として扱い、補正なし ($1.0$)
  - $10\% \le R_{\text{kage}} < 30\%$: 不整形地として軽微なマイナス補正 ($0.90 \sim 0.95$)
  - $30\% \le R_{\text{kage}}$ または区画図画像解析で「旗竿地（敷延）」と判定された場合: 旗竿地補正を適用 ($0.70 \sim 0.80$)

#### ② 間口狭小補正 ($C_{\text{width}}$) ＆ 奥行長大補正 ($C_{\text{depth}}$)
間口 $W_{\text{front}}$ および奥行 $D_{\text{depth}}$ に応じ、標準以下の使い勝手の土地にペナルティ係数を乗じます。
- **間口狭小**:
  - $W_{\text{front}} < 2.0\text{m}$: **再建築不可** (接道義務違反) ➔ 価値を極大減価 ($0.20 \sim 0.30$)
  - $2.0\text{m} \le W_{\text{front}} < 4.0\text{m}$: 間口狭小補正 ($0.85 \sim 0.95$)
- **奥行長大**:
  - 奥行が間口の2.5倍を超える場合、奥行長大補正 ($0.90 \sim 0.95$)

#### ③ 接道加算補正 ($C_{\text{road}}$)
角地や両面道路など、複数の道路に接している場合は価値の向上を反映します。
- **角地（側方路線影響加算）**: $1.03 \sim 1.08$ の加算
- **二方路（裏面路線影響加算）**: $1.02 \sim 1.05$ の加算

### 2.2 建築基準法（法規制）に基づく「実質容積率・消化力」の評価モデル

土地の価格は「そこにどれだけの床面積（ボリューム）を建てられるか」で決定されます。指定容積率がどれだけ高くても、以下の法規制により実質容積率が制限される土地は価値が下がります。

#### ① 前面道路幅員による容積率制限 ($V_{\text{road\_limit}}$)
建築基準法第52条に基づき、前面道路の幅員 $W$ に応じて容積率は以下のように制限されます。
*   **住居系用途地域**:
    $$V_{\text{actual\_limit}} = \min(V_{\text{specified}}, W \times 0.4 \times 100\%)$$
    *(例: 指定容積率200%であっても、道路幅員が4mの場合、住居系地域では $4 \times 0.4 = 160\%$ に制限されます。)*
*   **非住居系用途地域（商業・工業地域等）**:
    $$V_{\text{actual\_limit}} = \min(V_{\text{specified}}, W \times 0.6 \times 100\%)$$

#### ② 北側接道の有利性と斜線制限緩和モデル
建物は「道路斜線制限」「隣地斜線制限」「北側斜線制限」により、高さを出すと北側を削られる斜めカットの形状（斜線制限）を強いられます。

- **北側接道（北道路）の絶対的メリット**:
  北側斜線制限は「北側の隣地境界線」から立ち上がりますが、北側が道路である場合、制限の起点は **「道路の反対側の境界線」** に緩和されます。
  これにより、道路の幅（4m〜6m以上）だけ制限のスタートラインが外側に押し出され、結果として **敷地全体で斜線に干渉することなく、指定容積率の上限いっぱいまで箱型の最大床面積を確保できる可能性が劇的に高まります**。
- **アルゴリズムでの指標化 (実質容積消化力指数: $E_{\text{volume}}$)**:
  接道方位が「北」かつ「前面道路幅員が4m以上」である場合、斜線制限による容積消化阻害リスクが極めて低いと判定し、実質容積消化効率を $1.10 \sim 1.15$ の加算係数としてスコアに反映します。
  逆に、「南側接道」でかつ敷地の奥行きが短い土地は、北側斜線制限によってアットホームやアパートの3階部分が斜めに削られるか建築不能になるため、消化効率を $0.85 \sim 0.90$ にマイナス補正します。

### 2.3 開発収益還元法 ＆ 土地残余法（収益還元アプローチの高度化）

プロの仕入デベロッパーおよび不動産鑑定士は、単に「売出事例との比較」だけでなく、その土地に実際に建物を建築して運用（賃貸アパートなど）または分譲した場合の「収益性」から土地の投資価値を逆算する **「土地残余法（Land Residual Method）」** および **「開発法」** を適用します。

#### ① 土地残余法による土地価格の評価式
土地に最有効使用の賃貸アパートを新築すると仮定したときの、全体の純収益 $a$ から、建物に帰属する純収益 $a_B$ を差し引き、残った「土地に帰属する純収益 $a_L$」を土地の還元利回り $Y_L$ で資本還元して土地価値 $P_L$ を求めます。

$$P_L = \frac{a - a_B}{Y_L}$$
ここで、建物帰属純収益 $a_B$ は以下のように定義されます：
$$a_B = P_B \times (Y_B + d)$$
*   $P_B$: 想定建物建築価格（総工費）
*   $Y_B$: 建物の期待利回り
*   $d$: 建物の償却率（耐用年数に基づく定額減価償却費率）

#### ② 開発法シミュレーションモデル（デベロッパー仕入上限価格の算出）
1.  **想定ボリューム（最大建築可能床面積）の算出**:
    $$A_{\text{total\_floor}} = A_{\text{land}} \times V_{\text{actual\_limit}} \times E_{\text{volume}}$$
2.  **想定竣工総売上 (Gross Development Value: $GDV$) の推定**:
    該当エリアのアパート（または戸建て）の「平米あたり新築販売単価（または賃料還元価値）」を周辺のクローラーデータから算出し、想定床面積を乗じて GDV を算出します。
3.  **想定建築コスト・諸経費の算出**:
    構造（木造・RC造）に応じた平米あたり標準建築費（坪単価）から Hard Costs を算出し、設計費や融資金利等の Soft Costs を加算します。
4.  **仕入上限土地価格 ($V_{\text{land\_limit}}$) の逆算**:
    $$V_{\text{land\_limit}} = GDV - (\text{建築コスト} + \text{開発諸経費} + \text{デベロッパー想定利益})$$
    この $V_{\text{land\_limit}}$ と、実際の売り出し価格 $V_{\text{asking}}$ の乖離率を算出することで、**「事業用地としてどれだけ割安でお宝か」** を1発で自動スクリーニングできるエンジンとなります。

### 2.4 不動産鑑定士・プロ業者が組み込む「地盤・環境リスク ＆ 物理的要因」の減価モデル

路線価や事例比較の単純な数値だけでは見落とされ、実務で数百万〜数千万円の想定外の損失を引き起こす **「実務特有の減価要因」** をモデル化します。

#### ① セットバック（道路後退）減価の自動算出 ($C_{\text{setback}}$)
建築基準法第42条2項に基づき、前面道路の幅員 $W$ が 4.0m 未満（2項道路）の場合、道路中心線から 2.0m 後退（セットバック）した線が敷地境界とみなされ、セットバック部分は敷地面積から除外され、建築確認上の建ぺい・容積計算からも除外されます。
- **セットバック面積の簡易算出**:
  $$A_{\text{setback}} = \text{間口} (W_{\text{front}}) \times \frac{4.0 - W}{2}$$
- **減価評価**:
  有効土地面積を $A_{\text{land}} - A_{\text{setback}}$ に減じた上で、セットバック部分の価値を $0$ とみなして土地総額を自動減価します。

#### ② 高低差（がけ地）による造成工事費（土留め擁壁）控除モデル
高低差のある土地（がけ地・傾斜地）は、擁壁（ヨウヘキ）工事や造成（盛土・切土）を行わないと再建築できません。
- **がけ地減価の数式化**:
  $$V_{\text{land\_adjusted}} = V_{\text{land\_base}} - \text{想定造成費用} (C_{\text{development}})$$
  $$\text{想定造成費用} (C_{\text{development}}) = A_{\text{land}} \times \text{傾斜度別平米単価} + \text{擁壁長さ} \times C_{\text{yowheki\_unit}}$$
  ※傾斜度別造成費および擁壁工事費（1mあたり15万〜20万円等）の基準単価をマスタデータ化して自動控除します。
- **がけ条例による建築制限**:
  傾斜地に擁壁がない、または古い石積みの場合は「がけ条例」により崖から一定距離（崖高の2倍等）の建築禁止エリアが発生し、有効敷地が激減するペナルティ（有効面積補正係数 $0.5 \sim 0.7$）を適用します。

#### ③ 環境・地下埋設物・法的隠れたリスク減価
- **土壌汚染リスク**: 過去の利用履歴（ガソリンスタンド、クリーニング店、めっき工場等）が検出された場合、土壌汚染調査および除去費用（概算 300万〜1,000万円）を土地価値から控除します。
- **埋蔵文化財包蔵地**: 都市計画情報から「埋蔵文化財包蔵地」に該当する場合、試掘調査（期間約1〜2ヶ月）および本発掘調査費用（自己負担リスク）を考慮し、時間的遅延および金利ペナルティを含めたリスク割引率（$5\% \sim 10\%$ 減価）を乗じます。
- **文化財・土壌汚染等のマスタデータベース連携**:
  国土交通省のハザードマップや自治体の都市計画情報から取得できる情報を評価に紐づけます。

## 3. 実装に向けたロードマップ ＆ 戦術的アプローチ

開発工程を確実に成功させるため、実効性の高い「戦術（Tactics）」と、3つのフェーズに分けた詳細な実装ロードマップを定義します。

### 3.1 戦術的アプローチ (Tactics)

1.  **カバレッジ最大化のためのポータル優先順位付け**:
    一棟アパート・戸建ての推定精度向上において、最もサンプル数が豊富で「地主直の売地情報」が集まる **「アットホーム」および「LIFULL HOME'S」を最優先** で収集拡張します。その後、大手4社（三井・住友・東急・野村）の自社売地情報を順次統合します。
2.  **地価公示・都道府県地価調査データのマスタ化 (ベースライン確立)**:
    土地単体の適正価格を推定する基準値（ベースライン）として、国土交通省が公開している「地価公示」および「都道府県地価調査」のオープンデータをデータベースへインポートするバッチコマンド（マスタ化スクリプト）を作成します。クロールした土地情報の「住所（町丁目）」レベルで自動突合し、エリア基本単価を自動付与する戦術をとります。
3.  **セットバック・法規制判定の自動エンジン化**:
    前面道路幅員 $W < 4.0\text{m}$ の物件に対し、自動的にセットバック面積を算出し有効土地面積を補正する法規制スクリーニングアルゴリズムを最優先で開発します。
4.  **画像解析API予算管理と段階的機能展開**:
    Gemini API（Vision）の予算枠（1日200件）を最適化するため、以下の2段階アプローチを実装します：
    - **Stage 1 (テキスト解析)**: 間口や道路幅員などのテキストデータのみで一次価格比準シミュレーションを実行。
    - **Stage 2 (画像解析)**: 一次シミュレーションで「割安度が高い」と判定されたスクリーニング合格物件に限定して区画図（不整形・旗竿地判定）および外観写真を Gemini Vision に送信し、精密評価を行うことで、APIコストを最小限に抑えながら高精度な補正を実現します。

### 3.2 実装ロードマップ (Roadmap)

```
[Phase 1: 基盤整備と収集拡張]
 ├─ Tochi モデルへの属性追加・マイグレーション
 ├─ 公示地価・地価調査データインポートマスタの実装
 └─ アットホーム・ホームズ「売地」クローラーの新規実装 (WAF回避 header 適用)
        │
        v
[Phase 2: 土地評価アルゴリズム・シミュレータ実装]
 ├─ 道路幅員・指定容積率の判定ロジック開発
 ├─ 接道状況（北側接道緩和・角地補正など）の自動判定ロジック開発
 ├─ セットバック・がけ地造成費の自動計算エンジンの開発
 └─ 想定アパート建築プランに基づく「仕入開発法・土地残余法」シミュレータの実装
        │
        v
[Phase 3: 画像解析統合 ＆ 統合価格推定エンジンの結合]
 ├─ Gemini Vision による区画図の不整形地・gake条例影響の判定モデル結合
 ├─ 環境リスク（土壌汚染・埋蔵文化財包蔵地）データ連携の実装
 ├─ 土地推定価格を戸建て・アパート評価エンジンへ特徴量（Feature）として結合
 └─ 土地残余価値モデル of 適用による、最終予測理論価格の精度検証
```

#### フェーズ1: 収集基盤とベースライン確立
*   **タスク**: `package/models/tochi.py` 等への間口・幅員・道路方向等のフィールド追加、公示地価インポーターの実装。
*   **検証**: 一般的な住所で地価マスタが引けるか。アットホーム「売地」一覧の動的詳細URLがWAFを回避して200 OKで取得できるか。

#### フェーズ2: 法規制シミュレータおよび評価コア実装
*   **タスク**: 都市計画用途地域に基づく実質容積率シミュレータ、セットバック面積算出ロジック、造成費控除および土地残余法コアロジックの構築。
*   **検証**: 前面道路幅員（例: 3.5m ➔ セットバック発生）や接道方向を入力し、有効床面積や造成控除額が正しく計算されるかを検証するユニットテストの作成。

#### フェーズ3: 画像解析連携と統合MLモデルのデプロイ
*   **タスク**: 区画図からの旗竿地・不整形地の画像解析と Stage 2 精密判定の統合。土地推定価格をアパート・戸建て評価特徴量へ組み込み。
*   **検証**: 最終推定エンジンの RMSE（平均二乗誤差平方根）が、土地評価指標の導入前後で有意に改善されるかのテスト実行。
