# ドキュメント管理ガイドライン

## 目的

本ドキュメントは、`docs/` フォルダ内のドキュメントと `README.md` の整合性を保つためのガイドラインを定めます。

## ルール

### 1. ドキュメント追加時

`docs/` フォルダに新しいドキュメントを追加した場合、以下を実施すること：

1. **README.mdの「ドキュメント一覧」セクションを更新**
   - ドキュメント名とリンクを追加
   - 適切な分類（入門ガイド・技術仕様書・開発ガイド・運用）に配置
   - 簡潔な説明を記載

2. **分類の基準**
   - **🚀 入門ガイド**: 初めて使う方向けのドキュメント
   - **📐 技術仕様書**: システムの設計・仕様に関するドキュメント
   - **🛠️ 開発ガイド**: 開発者向けの実装・改善に関するドキュメント
   - **🔧 運用・トラブルシューティング**: 運用時の問題解決・レポートに関するドキュメント

### 2. ドキュメント修正時

既存のドキュメントを大幅に修正した場合：

1. **README.mdの説明を更新**
   - ドキュメントの内容が変わった場合、説明文を最新の内容に合わせる

2. **分類の見直し**
   - ドキュメントの性質が変わった場合、適切な分類に移動

### 3. ドキュメント削除時

`docs/` フォルダからドキュメントを削除した場合：

1. **README.mdから該当行を削除**
   - 「ドキュメント一覧」セクションから該当するドキュメントのエントリを削除

### 4. サイト構造ドキュメント

`docs/site_structures/` 配下のドキュメントを追加・削除した場合：

1. **README.mdの「🏢 サイト構造ドキュメント」セクションを更新**
   - 対象サイトと説明を追加・削除

## チェックリスト

ドキュメント作業後、以下を確認すること：

- [ ] `docs/` フォルダの内容とREADMEの「ドキュメント一覧」が一致している
- [ ] 各ドキュメントの説明が最新の内容を反映している
- [ ] 分類が適切である
- [ ] リンクが正しく機能する

## 例

### ドキュメント追加の例

`docs/deployment_guide.md` を追加した場合：

```markdown
### 🛠️ 開発ガイド

| ドキュメント | 説明 |
|------------|------|
| [development_guide.md](docs/development_guide.md) | 開発者向けガイド（環境構築、デバッグ方法、新規パーサー追加手順） |
| [deployment_guide.md](docs/deployment_guide.md) | デプロイメントガイド（本番環境へのデプロイ手順） |  <!-- 追加 -->
| [refactoring_suggestions.md](docs/refactoring_suggestions.md) | リファクタリング提案（コード改善のベストプラクティス） |
```
