# Field Name Unifier

パーサーの実装やモデル（`package/models/*.py`）の設計・変更を行う際に、フィールド名の統一基準を確保するためのルールです。

## 手順

1. **命名基準の確認**
   - サイトが異なっていても、同じ情報を保持する項目（例：バルコニー面積、備考など）は、モデル定義およびパーサー実装において必ず同一のフィールド名を使用すること。
   - 実装前に必ず `docs/internal_design/field_naming_standards.md` を参照し、フィールド名が標準化された命名規則に準拠しているかを確認すること。

2. **既存定義との照合**
   - 新規フィールドを追加する前に、既存モデル（他サイトの同物件種別のモデル等）ですでに同じ意味のフィールドが定義されていないかを検索し、重複や表記ゆれを防ぐこと。
