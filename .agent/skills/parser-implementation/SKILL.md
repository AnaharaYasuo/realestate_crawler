# Parser Implementation Procedure

新しいサイトのパーサーを開発、修正、または検証する際に、一貫した品質を保つための必須のワークフローです。全ての段階において `docs/implementation/parser_implementation_procedure.md` に規定された手順に厳格に従ってください。

## 手順

1. **仕様定義と実装確認**
   - サイトが異なっていても、同じ情報を保持する項目はモデル定義およびパーサー実装において必ず同一のフィールド名を使用すること。
   - 詳細は `docs/internal_design/field_naming_standards.md` を正として遵守すること。
   - パーサーの実装（ラベル起点抽出、フォールバックの扱い、検証プロセス等）はガイドラインに従うこと。

2. **検証プロセス (API実行時の必須タスク)**
   - 修正や実装を行った後は、対象となるサイトと物件種別ごとに正常に処理されるかを必ず検証すること。
   - **全サイト全物件種別** でデータが取得できるようにすること。
   - エラーHTMLが発生した場合は、そのHTMLを検証用として取得し、それを基に修正を当てること。
   - 不測の事態や検証時には、ログに十分詳細なエラー情報が出力されるように設定すること。
