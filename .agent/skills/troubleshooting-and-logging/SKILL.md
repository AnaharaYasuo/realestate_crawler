# Troubleshooting and Logging

意図しない問題やエラーが発生した際のデバッグ、原因究明、およびログ拡充についての必須プロセスです。

## 手順

1. **仮説構築と検証の徹底**
   - 問題が発生した原因に対して仮説を構築した場合、その仮説が正しいことを必ずログや実行結果で裏付け・確認してから原因対処（コードの修正等）を行うこと。
   - 単なる仮説のみに基づいて修正を行うことは禁止する。

2. **事実の収集とログ拡充**
   - 問題の特定には事実の収集が重要であるため、ログに出力する情報は十分かつ詳細なものにすること。
   - 既存のログで問題特定に必要な情報が不足していると判明した場合は、修正作業の一環として必ずログに出力する情報を拡充（適切なエラー出力や変数のダンプを追加）すること。

3. **一時ファイルの管理 (Tempフォルダ制約)**
   - デバッグスクリプト、再現用HTML、DOM確認ツールなど、一時的に使用する作業用のコードや出力ファイルは、必ず `Temp` フォルダに作成・保存すること。
   - 作業終了後は `Temp` フォルダ内に留めるか、不要であれば削除してプロジェクト直下を汚さないようにする。

4. **ルールと問題解決の自己反映**
   - 繰り返しユーザーから同じ指摘を受けた場合、それは重要な指示であると判断し、プロジェクトのルール（または関連するワークフロードキュメント）として記録・保存すること。
   - 起こした問題の解決方法（再発防止策）についても、都度関連ドキュメントに追記・反映させること。
