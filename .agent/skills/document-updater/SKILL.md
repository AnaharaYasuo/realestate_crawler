# Document Updater Skill

このスキルは、コードやデータベーススキーマに変更があった際に、関連するドキュメントを漏れなく更新するための手順です。

## 手順

1. **`implementation_plan.md` / `walkthrough.md` の作成・更新**
   - これらのファイルは **必ず日本語** で出力すること。

2. **`docs/` 配下のドキュメント参照**
   - 実装や設計の際は、`docs/` 配下のドキュメント（仕様書、DBスキーマ、サイト構造など）を常に参照し、最新の仕様に準拠すること。
   - コードを修正したら、必ず関連する `docs/` 内のファイルや `README.md` を最新のコードに合わせて修正すること。

3. **モデル定義変更時の同期**
   - `package/models/*.py`（モデル）を変更した場合、フィールドの追加・削除・型変更・NULL制約変更など、すべてのモデル変更を **必ず** `docs/database_schema.md` に反映すること。

4. **`README.md` の「ドキュメント一覧」更新**
   - `docs/` フォルダにドキュメントを追加・修正・削除した場合、**必ず** `README.md` の「ドキュメント一覧」セクションを更新すること。
   - 更新内容：ドキュメント名とリンクの追加、適切な分類（🚀入門ガイド・📐技術仕様書・🛠️開発ガイド・🔧運用）への配置、簡潔な説明の記載。
   - 詳細は `docs/documentation_guidelines.md` を参照すること。
